Artist-first foundation

A focused first step for local image generation inside Photoshop.

OpenLayer starts with one dependable workflow: text-to-image generation from Photoshop to ComfyUI, a panel preview, and import into the active document. The goal is a clean base that can grow into deeper layer, mask, and selection workflows without rushing fake features.

MVP features

What works now

ComfyUI connection

Set a local server URL, check availability, and load available checkpoints from ComfyUI.

Text-to-image workflow

Send prompt, negative prompt, size, steps, CFG, seed, and checkpoint into a starter workflow.

Panel preview

Retrieve generated output through ComfyUI history and view endpoints, then show it in the panel.

Photoshop layer import

Place the generated image into the active document and rename it with an OpenLayer timestamp.

UXP-safe interface

Built with vanilla TypeScript and static UI code that avoids browser APIs missing in Photoshop UXP.

Future-ready structure

ComfyUI, Photoshop, UI, workflow, and utility modules are separated for careful next steps.

The MVP loop in Photoshop

Workflow

How it works

  1. Open Photoshop and load the OpenLayer UXP panel.
  2. Run ComfyUI locally on the OpenLayer server port.
  3. Choose a checkpoint and enter a prompt.
  4. Generate through the ComfyUI prompt API.
  5. Preview the output image in the panel.
  6. Import the result as a new named Photoshop layer.

Setup

Run the MVP locally

1

Install and build

npm install
npm run build
2

Load in UXP Developer Tool

Select the generated dist/manifest.json, click Load, then open the OpenLayer panel in Photoshop.

3

Start ComfyUI

python main.py --listen 127.0.0.1 --port 8190
4

Create the first layer

Click Check ComfyUI, choose a checkpoint, generate, preview, and import the result into the active document.

OpenLayer defaults to http://127.0.0.1:8190 so it can run beside other local tools that may already use port 8188.

Known limitations

Honest v0.1 boundaries

  • Only the txt2img-basic workflow is supported right now.
  • Custom ComfyUI workflows may need node ID updates in the workflow builder.
  • Generated images are placed as Photoshop layers; advanced alignment is future work.
  • No selected layer export, image-to-image, inpainting, masks, or ControlNet-style tools yet.
  • The browser dev server is only for UI preview. Photoshop APIs run inside UXP.
